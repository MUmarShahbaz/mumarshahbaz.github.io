Laser Security System

Difficulty: A

Self-Learning Projects for Computer Engineering

Demo

Explanation

The apparatus is set up so that an LDR module will send live status of whether the laser is present or not. The reason why an alarm can’t be directly attached is that as soon as the intruder has passed through, the laser will once again get in contact with the LDR and it will stop sounding the alarm. This means that the alarm will only sound for approximately 1s which is not an appropriate duration for a security alarm. To counter act this I am using a bits to store 1 when a laser is broken and 0 when manually set to via a pushbutton. For this I used a 5 Bit Memory Unit—a previous project of mine, a link is given below at the end of the page—and by assigning each laser to a separate but, we can also store exactly which laser was broken as can be seen above in the demo video.
